# WebOCR

Распознавание английского текста с изображений (разные шрифты и размеры)
прямо в браузере. Модель обучается с нуля, обучение и сборка полностью
автоматизированы через GitHub Actions.

## Архитектура

- **Модель**: CRNN (CNN + BiLSTM) + CTC-loss — стандартная и лёгкая архитектура
  для распознавания строк текста. Без квантизации весит десятки МБ, легко
  укладывается в лимит 100 МБ (при желании ещё сильнее ужимается через
  `torch.quantization` до int8).
- **Обучение**: PyTorch, синтетические данные — рендерим случайный текст
  разными шрифтами/размерами/шумом (`training/generate_dataset.py`), поэтому
  модель с самого начала видит разнообразие шрифтов.
- **Инференс**: Rust + [`tract`](https://github.com/sonos/tract) — чистый Rust
  ONNX-движок без C/C++ зависимостей, что критично для стабильной сборки под
  `wasm32-unknown-unknown` (в отличие от `onnxruntime`, который тяжело собирать
  под wasm).
- **Сборка**: `wasm-pack build --target web`.

## CI/CD: два независимых workflow

### `train.yml` — обучение модели
Запускается:
- вручную (Actions → Train Model → Run workflow), можно задать число
  синтетических примеров и эпох;
- автоматически при пуше изменений в `training/**`.

Что делает:
1. Ставит системные английские шрифты (`fonts-dejavu`, `fonts-liberation`,
   `fonts-noto-core`, `fonts-freefont-ttf` и т.д.) — этого достаточно для
   разнообразия стилей без ручной загрузки шрифтов.
2. Генерирует синтетический датасет (`generate_dataset.py`).
3. Обучает CRNN (`train.py`) и экспортирует в ONNX (`export_onnx.py`).
4. Публикует `model.onnx` и `crnn_ocr.pt` как артефакт сборки.
5. Коммитит обновлённый `rust/assets/model.onnx` обратно в ветку `main`.

Этот коммит автоматически запускает `build.yml`.

⚠️ **Важно про ресурсы CI**: стандартные раннеры GitHub Actions — это CPU
без GPU, максимум ~6 часов на джобу (public-репозитории). Полноценное
обучение (50k+ примеров, 20+ эпох) на CPU может не уложиться по времени.
Для реальных экспериментов:
- уменьшите `samples`/`epochs` во входных параметрах workflow_dispatch
  (дефолты — 20000 примеров / 15 эпох, это уже компромисс под CI);
- либо подключите self-hosted раннер с GPU для более серьёзного обучения,
  просто заменив `runs-on: ubuntu-latest` на `runs-on: self-hosted` в
  `train.yml`.

### `build.yml` — сборка WASM
Запускается при пуше в `main` (кроме изменений внутри `training/**`, чтобы
не пересобирать wasm на каждый коммит с данными обучения) и после успешного
завершения `train.yml`. Делает:
- `wasm-pack build --target web --release`,
- `cargo test`,
- проверку, что итоговый `.wasm` не превышает 100 МБ,
- деплой `www/` на GitHub Pages.

## Локальный запуск (без CI)

### 1. Данные и шрифты
```bash
cd training
pip install torch pillow onnx
# при желании добавьте свои .ttf/.otf в training/fonts/
python generate_dataset.py --n-samples 20000
```

### 2. Обучение
```bash
python train.py --epochs 15        # сохранит crnn_ocr.pt
python export_onnx.py              # экспортирует в rust/assets/model.onnx
```

### 3. Сборка WASM
```bash
cd ../rust
cargo install wasm-pack            # один раз
wasm-pack build --target web --release
```

### 4. Демо в браузере
```bash
cd ../www
python -m http.server 8080
# откройте http://localhost:8080
```

## Возможные улучшения
- Заменить greedy CTC-декодирование на beam search для точности на
  неоднозначных символах.
- Квантизация модели (`torch.quantization`) для уменьшения размера в 2-4 раза.
- Добавить детектор строк текста перед распознаванием, если на входе может
  быть не одна строка, а целая страница/фото документа.
- Кэшировать сгенерированный датасет между запусками `train.yml`
  (`actions/cache`), чтобы не генерировать заново при каждом обучении.
